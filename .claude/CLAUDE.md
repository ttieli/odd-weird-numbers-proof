# RavenThought（渡鸦）

> 这个项目是为了帮助我梳理每天的待办事项。

## 核心规则

**每次对话都必须产出文档。** 不论用户发什么内容——哪怕只是打招呼、闲聊、一句话——都要记录到当日主文档中。没有例外。

- 每天一个日期目录：`daily/YYYYMMDD/`（短日期格式，无横杠）
- 主文档：`YYYYMMDD_每日汇总.md`，当天文件已存在则追加/更新，不覆盖已有条目
- 有深度内容的课题创建子文档，放在 `topics/` 子目录下，主文档中简述并关联链接
- 简短内容（打招呼、零散想法、简单问答）直接写入主文档，无需创建子文档

## 文档架构

```
RavenThought/
├── .claude/                              # Claude 配置和记忆
└── daily/                                # 所有待办和课题
    └── YYYYMMDD/                         # 日期目录（短格式）
        ├── YYYYMMDD_每日汇总.md           # 当日主文档
        └── topics/                       # 课题子文档目录
            ├── YYYYMMDD_课题A.md
            └── YYYYMMDD_课题B.md
```

### 命名规范

`daily/` 内**所有**文档统一使用 **`YYYYMMDD_内容描述.md`** 格式：

- 日期采用短格式 `YYYYMMDD`（无横杠），如 `20260302`
- 内容描述**优先使用中文**，简洁 2-6 个字
- 主文档：`YYYYMMDD_每日汇总.md`，如 `20260302_每日汇总.md`
- 课题子文档：`YYYYMMDD_内容描述.md`，如 `20260302_AI浏览器自动化.md`
- 英文专有名词保留原文，如 `20260302_Obsidian笔记自动化.md`
- 目录名也用短日期：`daily/20260302/`

### 课题子文档（`topics/`）

- 放在日期目录下的 `topics/` 子目录中
- 主文档通过 `./topics/YYYYMMDD_内容描述.md` 相对链接引用
- 每个文档顶部标注来源链接和关联项目路径
- 内容为某个课题的深度笔记、调研摘要等

### 外部链接内容处理规则

收到小红书、微信公众号或其他外部链接/截图时，**必须获取完整内容（含图片 OCR）并追溯信息源**：

1. **获取原文**：用 `wf "链接" --stdout` 抓取完整 Markdown（含图片链接）；截图用 `macocr "图片路径" --stdout` OCR
2. **追溯信息源**：如果文章引用/翻译了外部原文（论文、文章、新闻等），必须用 `wf` 抓取**原始信息源全文**。如果原文超长（如整本书、超过 50 页的 PDF），保留原文下载地址即可，不必全文粘贴
3. **创建 topic 文件**：在当日 `topics/` 下创建子文档
4. **文档结构**（按阅读优先级排列）：
   - 顶部：来源链接、原文链接、日期、关联项目路径
   - `## 总结`：提炼核心观点、关键信息、行动建议（3-5 条）
   - `## 关联`：与用户已有项目或其他待办的关联分析
   - `## 微信/小红书原文`：直接粘贴 `wf` 输出的完整 Markdown（不做改写）
   - `## 引用内容中文翻译`：如原文为外文，此处放中文翻译版本（微信文章本身即翻译时可注明"见上方微信原文"）
   - `## 引用内容原文`：信息源的原始语言全文（用 `wf` 抓取）；无法抓取时注明原因并保留 URL
5. **更新主文档**：在当日主文档中添加简述条目 + topic 链接

## 项目记忆

### 对话开始时

1. 读取 `.claude/project-memory.md`（如存在）
2. 扫描 `.claude/sessions/*.md`（未合并的会话）
3. 按时间排序，重点关注最新记录
4. 简要告知用户已了解的上下文（一句话即可）
5. 归档子文件（`.claude/archive/*.md`）通常不读，除非用户明确要查历史

### 对话结束时

当用户说"再见"、"bye"、"结束"、"保存进度"时：
- 使用 `/project-memory` skill 保存会话

### 自动保存

满足以下任一条件时，自动调用 `/project-memory` 保存当前进度（静默执行，保存后简短提示一句）：
- 完成了一个明确的功能或修复
- 对话轮数达到 20 轮
- 用户开始一个新的、不相关的任务
- 收到 context 压缩警告时（优先于压缩执行）

### 其他命令

| 用户说 | 动作 |
|--------|------|
| 合并记忆 / merge sessions | 合并所有 session 到主记忆文件 |
| 查看会话 / list sessions | 列出所有未合并的会话 |
