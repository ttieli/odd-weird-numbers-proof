## 总结

1. **科技公司经营不可持续**：人月单价 2.8 万 vs 成本 5 万，严重成本倒挂，集采模式无法改变
2. **改革方案**：公司转型为成本中心，留 100 人挂集团科技研发中心牌子；486 人+约 20 多名派遣制同事整体回光大银行
3. **时间节点**：预计 3 月 31 日签约（先解约科技公司合同，再签银行合同），过渡期 3-6 个月
4. **薪酬**：回行/留下均按对应体系"入位"，同岗同酬，月度绩效浮动调整；回行后工资结构变为 5:2:3（工资:季度绩效:年终）
5. **部门情况**：本部门留 18 人组成"数据处"，其余回银行；数据服务团队整体回去，平台团队留 10 人

## 一、为什么要改——经营困境

| 项目 | 现状 |
|------|------|
| 新项目人月单价 | 2.8 万元 |
| 实际人均成本 | 5 万元 |
| 差距 | **成本倒挂近一倍** |

- 2018 年至今做出一些成果，但经营模式问题始终未解决——做一单亏一单
- 去年尝试"打开后端"：清理验收存量项目
- 也想"解决前端"：按成本价（5 万/人）直签集团或银行合同，但**集采模式无法改变**（集团改不了，银行改不了，集采审查严格）
- 最终方案：**改变人员结构，全面转型，推进成本中心化**

---

## 二、怎么改——改革方案

### 2.1 留下的人（~100 人）

| 项目 | 内容 |
|------|------|
| 总人数 | 集团批复最多 **100 人** |
| 组织架构 | 挂牌"**集团科技研发中心**"，下设 **5 个处** |
| 五个处 | 数据处、开发处、综合处、运维安全处、数字化办公处 |
| 本部门留下 | **18 人** → 归入数据处 |
| 总经理 | 大约在国际（兼集团科技部副总） |
| 重点工作 | 运维、安全、数据穿透、数据治理 → 保障金控集团管控 |

- 最终名单今天（3/25）已定：**93-94 人**
- 留下标准：做过集团项目、年龄不超龄、考核达标，经多轮筛选

### 2.2 回银行的人（486 + 20 余名派遣制）

| 项目 | 内容 |
|------|------|
| 回行人数 | **486 人** + 约 20 多名派遣制同事 |
| 接收方式 | **整体接收**，在银行研发中心下成立虚拟部门，先切入该部门 |
| 过渡期 | **半年**（实际可能 3 个月即可到各部门就位） |
| 可能去向 | 数据资产管理部、研发中心、智能运营中心等金科条线部门 |
| 派遣制 | 先切为银行派遣制，签约后"择优录取"（郝行长会上表态） |

- 公司班子（如领导与张勇的任命体系）过渡期内继续保留，确保项目平稳过渡
- 回行后与原行员**一视同仁**：可参加竞聘主管、招贤馆等机会，无区别对待
- 需重签合同

### 2.3 签约流程与时间

- **预计 3 月 31 日**执行
- 流程：先签科技公司**解约合同** → 再签银行**签约合同**
- 回行人员稍后会收到一本厚的纸质材料，需填写**干部履历表**（入行传统流程）

## 三、薪酬与职级

### 3.1 总体原则

- **入位制**：按现有薪酬刷入对应体系
- **同岗同酬**：定级后该级别对应固定薪酬框架
- 偏低者可能微涨，偏高者可能微降，部门整体在合理范围内
- 留下 vs 回去薪酬基本一致，测算留下的可能**略高一点点**

### 3.2 银行薪酬结构

| 组成 | 占比 | 说明 |
|------|------|------|
| 工资 | **50%** | 固定部分 |
| 季度绩效 | **20%** | 按季发放 |
| 年终奖 | **30%** | 年底发放 |

- 回行后下个月工资会**明显减少**（因结构变化，非实际降薪）
- 党费只算工资部分（比原来公司 8:2 体制有利）
- 原公司体制：工资 + 年终奖 = 8:2

### 3.3 职级晋升（银行体系）

- 硕士应届生默认 **11 级行员**
- **3 年涨一大级**（表现不错的情况下）
- 每年根据考核定分，越往高级越难
- 举例：7 级升 6 级（主管级）需 3 年内至少 **2 个 A**；低级别升级门槛较低（如 3 年 B+ 即可）
- **认前三年考核结果**（银行体系三年一更新，公司断档问题导致此前多年未涨薪）
- 工龄认可，行龄不认（无行龄积累）

## 四、本部门具体安排

### 4.1 人员分配

| 去向 | 人员 | 说明 |
|------|------|------|
| **留下（数据处）** | 18 人 | 平台团队留 10 人 |
| **回银行** | 其余全部 | 数据服务团队整体回去 |

### 4.2 项目影响

| 项目 | 影响评估 |
|------|----------|
| 外部数据项目 | **平稳**（数据服务团队整体回去，不影响） |
| 数据湖项目 | **有一定影响**（平台团队只留 10 人） |

- 铁力仍为平台团队主管，海刚为服务团队主管，后续再整体协调
- 银行设备项目刚中标，还需外采，过渡期继续推进

### 4.3 回行后的"起跑优势"

- 数据资产管理部对科技公司回行人员较为器重
- 孙总明确表态：科技公司回去是**等建制**接收
- 可能会有组织架构上的安排（植树、建团队）
- **但优势只是起跑优势**，仍需做出项目价值和业绩才能兑现

## 五、领导定调与要求

### 5.1 石总定位

> 此方案是**最优解**。极端方案（裁员）杀伤力太大，当前方案保障了公司延续和员工稳定。

### 5.2 郝行长贡献

- 认司龄、无固定期限合同等政策都是**郝行长最终拍板**
- 行长对科技公司"先骂后认可"，最终推动了整体接收方案
- 整建制回行、成立虚拟部门过渡——比直接打散回去好得多

### 5.3 心态调整

- 回行的人：需从**乙方心态**转为甲方/行员心态
- 留下的人：需调整原来的甲方心态
- 双方都要调整工作模式，做出价值

### 5.4 纪律要求

| 要求 | 说明 |
|------|------|
| **劳动纪律** | 关键时期严格遵守，不能违反 |
| **保密要求** | 严禁在小红书等社交媒体发布相关信息 |
| **系统安全** | 保障系统运行安全 |
| **项目交接** | 过渡期内支持好、配合好、过渡好 |

> 银行去年因年终奖信息在小红书泄露而非常被动，引以为戒。

## 六、展望

- 科技公司未来仍有扩张可能——如果领导诉求变化，可能再次扩编到五六百人
- 回行定位：**科技赋能、科技驱动**，把以前的优点发扬、不足弥补
- 留下定位：做好**穿透管控、亮点创新**
- 核心理念：**得到的就是最好的，做出价值来**

## 七、关键问答摘录

| 问题 | 回答 |
|------|------|
| 为什么只认三年考核？ | 银行体系三年一更新，这是制度规则 |
| 留下好还是回去好？ | 各有利弊，都需做出价值；得到的就是最好的 |
| 级别怎么定？ | 级别平移，同岗同酬入位 |
| 过渡期多久？ | 半年（实际可能 3 个月即到位） |
| 合同怎么签？ | 先解约后签约，3/31 执行 |
